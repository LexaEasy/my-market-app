# Архитектура седьмого спринта

## Цель

В седьмом спринте проект остаётся витриной интернет-магазина, но расширяется до Maven-мультипроекта с отдельным сервисом платежей и Redis-кешем товаров.

## Модули

- `market-app` — основное веб-приложение витрины интернет-магазина.
- `payment-service` — отдельный RESTful-сервис платежей на Spring WebFlux.
- `api` — каталог OpenAPI-спецификаций для интеграции сервисов.

## Порты

- `market-app` запускается на порту `8080`.
- `payment-service` запускается на порту `8081`.
- Redis запускается локально на порту `6379`.

## Redis

Redis используется как кеш товаров в основном приложении.

Выбранный способ запуска Redis: локальный Redis на `localhost:6379`.

Кешируются:

- список товаров;
- карточка товара.

При отсутствии данных в кеше приложение загружает товары из R2DBC-БД и сохраняет результат в Redis.

TTL кеша товаров: `2 минуты`.

## Интеграция с payment-service

Интеграция между `market-app` и `payment-service` описывается OpenAPI-спецификацией в `api/payment-api.yaml`.

По спецификации должны генерироваться:

- реактивный HTTP-клиент для `market-app`;
- серверные интерфейсы и модели для `payment-service`.

Минимальный контракт платежей:

- `GET /payments/balance` — получить текущий баланс;
- `POST /payments/pay` — выполнить оплату заказа и списать сумму.

Обмен между сервисами выполняется в формате JSON.

## Поведение покупки

`market-app` перед оформлением заказа обращается в `payment-service`.

Заказ создаётся только после успешного платежа.

Если баланс меньше суммы корзины, кнопка оформления заказа должна быть недоступна.

Если `payment-service` недоступен, кнопка оформления заказа должна быть недоступна, а пользователь должен увидеть сообщение об ошибке.

Если платёж не прошёл, заказ не создаётся.
